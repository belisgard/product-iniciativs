Маршрут инициативы по колонкам

МояКоманда (pre): Идея → Инициатива → Проверка на актуальность → Оценка востребованности → Оценка эффекта → Финансовый расчёт → 🚦 Очередь инициатив

МояКоманда · ⚙️ Создание (одна колонка в МК, внутри — полный конвейер Kaiten):
Kaiten: Проектирование → IT ресерч → Фин. оценка → Проектный комитет → Разработка → Интегр. тестирование → Сдача → Передача на сервис → Релиз → Готово

МояКоманда (post): Опытная эксплуатация → Промышленная эксплуатация → Обзор результатов → ✅ Завершено

Точки передачи между системами: МК → Kaiten после Gate A Go (карточка МК → «⚙️ Создание», карточка Kaiten → «Проектирование») и Kaiten → МК после релиза («Готово» Kaiten → «Опытная эксплуатация» МК).

🚦 Gate A — Go / No-Go

Отдельная колонка МК «🚦 Очередь инициатив» после финансового расчёта. Решение РП за 3 дня по 4 критериям: востребованность ≥60% ЦА, эффект ≥10% метрики или ≥100K₽/мес, окупаемость ≤12 мес, попадание в OKR. Go → карточка МК переводится в «⚙️ Создание», в Kaiten создаётся карточка в «Проектирование». No-Go → карточка МК в «Отклонённые идеи».

🗓 Комитет приоритизации (опционально, weekly). РП вправе собрать руководителей отделов и профильных директоров для коллегиального Go/No-Go и пересортировки очереди. Если РП уверен — решает единолично. Не путать с Проектным комитетом (этап 13, Gate B).

TO-BE улучшения процесса

🔍 Tech Lead review внутри колонки «Проектирование» Kaiten (30 мин, до упаковки ТЗ) — ранняя проверка технической реализуемости.

🛡 Security + Support checklist встроен в шаблон ТЗ как раздел «Операционная готовность» — вовлекаем ИБ и Support на этапе ТЗ, а не на релизе.

⏱ Нормативы длительности указаны на каждой колонке; в паспорте — «Журнал этапов» с полями Вход / Выход / Факт / Норма / Статус.
